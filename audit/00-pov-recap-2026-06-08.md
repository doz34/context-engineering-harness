## 3. Bugs found and fixed during POV

- **Bug 1** (`code_api.py`): `allowed_builtins=None` default value left `restricted_builtins` empty → NameError on `print`. Fix: explicit `if allowed_builtins is None: self.allowed_builtins = set(SAFE_BUILTINS)`.

- **Bug 2** (`code_api.py`): `getattr(__builtins__, name)` did not work in method scope. Fix: explicit `import builtins as _b`.

- **Bug 3** (`pii_tokenizer.py`): regex patterns missing comma, slash. Refined.

- **Bug 4** (`srs_linter.py`): regex `^AC-N` did not match with indentation. Fixed with `^\s*`.

- **Bug 5** (`hooks.py`): `post_tool_use_summarize_swallowed` also swallowed FAILED results. Fixed with `failure_markers` check.

- **Bug 6** (`subagent_firewall.py`): Tests were ill-written. Corrected to match real return contract.

## 6. Architecture decisions taken

| Decision | Choice | Justification |
|----------|--------|---------------|
| Language | Python 3.10+ | Rich stdlib, typer, sqlite native |
| State | SQLite WAL | Atomicity, queryable, portable |
| DSL | KEY:VALUE;;KEY:VALUE | Human-readable, machine-parseable, swebok-compatible |
| Hard cap behavior | ABORT | Avoid silent degradation |
| Subagent return | summary + refs + artifacts | Anti game-of-telephone |
| Compression ratio target | 10× | 80K → 8K measured |

---

## 7. Known risks

| Risk | Mitigation |
|------|-----------|
| SQLite lock contention in multi-agent | WAL mode, short transactions |
| Python 3.10 vs 3.11+ features | No 3.11+ features used for now |
| Codex/MCP integration | S2 - hooks SDK |
| Code-as-API not implemented | S3 - RestrictedPython sandbox |
